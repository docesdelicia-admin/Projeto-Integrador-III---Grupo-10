# Documentação — Projeto Integrador III

Esta pasta centraliza toda a documentação do projeto.

## Conteúdo esperado

- **Diagramas** — diagramas de caso de uso, classes, entidade-relacionamento (ER), sequência, etc.
- **Requisitos** — especificação de requisitos funcionais e não funcionais do sistema.
- **Anotações** — anotações de reuniões, decisões de arquitetura e registros de progresso.
- **Instruções de uso** — guias de instalação, configuração e uso do sistema para usuários finais e desenvolvedores.

## Estrutura sugerida

```
docs/
├── diagramas/        # Diagramas UML e ER
├── requisitos/       # Documentos de requisitos
├── reunioes/         # Atas e anotações de reuniões
└── guias/            # Guias de uso e instalação
```
